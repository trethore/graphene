## Required Splits

1. `GrapheneCore` -> common consumer/runtime registry plus Fabric bootstrap, lifecycle, mod resolution, and ID
   validation.
2. `GrapheneCoreServices` -> explicitly installed common runtime plus Fabric-provided platform services.
3. `GrapheneCefRuntime` -> common JCEF lifecycle/state machine plus injected Fabric lifecycle, main-thread, startup UI,
   and shutdown adapters.
4. `GrapheneBrowser` -> common internal off-screen browser and frame capture plus Fabric rendering, native-window, and
   cursor integration.
5. `BrowserSurface` -> common browser session/viewport concepts plus Fabric `GuiGraphics` surface API.
6. Dialog managers -> common internal CEF callback adapters plus Fabric screen/native-dialog presenters.
7. URL APIs -> common Graphene resource identifiers instead of `net.minecraft.resources.Identifier`.
8. Load listener APIs -> Graphene-owned load events instead of public CEF callback types.
9. Browser configuration -> Graphene-owned options instead of public `CefBrowserSettings` and `CefRequestContext`.

## Migration Order Derived from the Inventory

1. Configuration and portable value objects.
2. Resource identifiers, URL generation, MIME handling, and HTTP server.
3. Bridge API and protocol after introducing a JCEF-independent browser endpoint.
4. Runtime/platform service contracts and explicit installation.
5. Internal JCEF runtime and browser capture implementation.
6. Fabric bootstrap, rendering, input, screens, dialogs, overlays, and mixins.
7. Debug source set and manual test pages.

## Completion Rule

A row moves from `Pending` to `Migrated` only after its destination compiles, associated tests pass, old behavior is
accounted for, and no forbidden dependency crosses the common/Fabric boundary.
